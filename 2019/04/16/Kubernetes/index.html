Dockerfile常用命令

  • FROM ：指定基础镜像。

  • MAINTAINER: 指明维护者信息。

  • RUN: 执行跟随的命令。

  • EXPOSE: 暴露容器的端口号。可以在启动容器时通过-p 指定本地端口和容器端口的映射。

  • WORKDIR： 配置工作目录，后续的RUN、CMD、ENTRYPOINT将工作在该命令指定的目录下。可以使用多个WORKDIR指令，后续命令如果参数是相对路径，则会基于之前命令指定的路径。

  • VOLUME: 创建一个可以从本地主机或其他容器挂载的挂载点，一般用来存放数据库和需要保持的数据等。

  • ENV: 指定环境变量，该环境变量会被后续RUN指令使用，并在容器运行时保持。

  • ADD：复制指定的src到容器中的dest。src可以是Dockerfile所在目录的一个相对路径（文件或目录），也可以是一个URL，还可以是一个tar文件（自动解压为目录）。

  • COPY: 复制本地主机的src为容器中的dest，src为Dockerfile所在目录的相对路径，文件或者目录，当目标路径dest不存在时，会自动创建。COPY是ADD指令的简化版，当需求是复制文件或目录到容器中时，推荐使用COPY指令。

ADD和COPY的区别：ADD指令的功能比COPY功能复杂。ADD指令中，src可以是URL，此时将根据URL进行下载，下载的文件放在dest目录；src还可以是tar文件，此时将自动解压到dest目录。但是下载和解压功能不可同时生效。而COPY只是复制功能。

  • CMD: 指定启动容器时执行的命令，每个Dockerfile只能有一条CMD命令，如果指定了多条CMD命令，只有最后一条会被执行。另外，如果用户启动容器时指定了运行的命令（docker run），则CMD指定的命令会被覆盖掉。

  • ENTRYPOINT: 配置容器启动后执行的命令，并且不可被docker run提供的参数覆盖。每个Dockerfile中只能有一个ENTRYPOINT，当指定多个时，只有最后一个生效。

CMD 和ENTRYPOINT的区别：CMD 指定的命令可在启动容器时被覆盖，而ENTRYPOINT指定的命令则不会。ENTRYPOINT更加灵活，它的Exec格式和shell格式效果差别大。使用Exec格式时，可以通过与CMD一起使用，同时提供不可变和可变的参数；而shell格式会忽略任何CMD或docker run提供的参数。例如下面的Dockerfile片段：
